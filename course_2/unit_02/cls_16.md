**Site hierarchy best practices**
## 网站层级结构的最佳实践

合理规划网站层级结构，可以让 Google 搜索引擎更容易抓取并索引你的网站页面。抓取是指发现新的或已更新页面的过程。良好的网站层级结构还能确保 URL 简洁易读，提升 SEO 表现，并让用户更容易在你的网站上找到所需内容。本节将帮助你了解网站层级结构的最佳实践。

**规划网站层级结构**

每个网站都有一个首页，也称为根页面。这是构建网站层级结构的起点，也是许多用户首次访问你的网站时进入的页面。

首页位于网站层级结构的最顶层，通常应包含有关企业或产品的概括性信息。位于首页之下的子页面（即较低层级的页面）则应提供更具体的信息，例如产品详情。

网站层级结构通常包含三到四个层级。每一层的信息都会比上一层更加具体。有些网站的层级超过四层，但一般建议让每个页面距离首页不超过三到四次点击。

假设你在一家在线书店工作。该书店的首页通常会展示大多数用户感兴趣的内容，例如当前优惠、新书推荐和畅销书。

首页之下的第一层通常是分类页，用于将相似的内容或产品进行分组。例如，在线书店的分类可能包括小说、非小说类和儿童读物。

第二层是子分类页，与分类页相关联。例如，在儿童读物分类下，网站可能会进一步细分为章节书、绘本、漫画、套装书等。

第三层则是更具体的页面，例如某一本儿童书籍的产品详情页。

**使用面包屑导航**

假设一位访客进入了某本儿童书的产品详情页，但想继续浏览其他书籍，该如何返回更大范围的书籍列表？一种常见方式是使用面包屑导航。

面包屑导航是位于页面顶部或底部的一组内部链接，帮助访客快速返回上一级页面或首页。

面包屑通常从首页开始，依次展示通向当前页面的网站层级路径。

**使用简单、描述清晰的 URL**

网站层级结构的另一个最佳实践是创建简洁且具有描述性的 URL。URL 中的文字应能清楚说明该页面的内容，方便访客理解点击后会看到什么，同时也有助于搜索引擎理解页面主题，从而提升 SEO。

**URL 的最佳实践**

尽量保持 URL 简短。
过长的 URL 可能会让访客感到困惑或不知所措。

较短的 URL 也更方便访客分享。

让 URL 易于理解。
访客在点击 URL 时，应能大致判断链接将带他们去哪里。

例如，如果你正在为“悬疑小说”这一子分类页面创建 URL，可以使用：
[https://www.booksgalore.com/fiction/mystery](https://www.booksgalore.com/fiction/mystery)
访客可以清楚地知道这是一个与书籍相关的网站，并且该链接指向小说分类下的悬疑小说页面。（注意：Books Galore 为虚构网站，链接不可用。）

URL 中应使用有意义的文字，而不是对访客毫无意义的数字。同时，应避免使用诸如“page1”之类的泛化词语，或在 URL 中堆砌关键词，例如 “books-fiction-books-fictionbooks”。

请记住，URL 会显示在搜索引擎结果中，因此务必确保其对用户友好。

**保持 URL 的更新**

当网页被删除或内容被移动到其他位置时，可能会产生断链。断链是指指向不存在页面的链接。

**创建有用的 404 页面**

有时访客会进入 404 页面，这类页面用于告知用户他们尝试访问的网页不存在。

如果 404 页面只显示模糊的信息，或没有任何后续指引，访客往往会无从下手。

自定义的 404 页面可以引导访客返回网站中仍可正常访问的页面，从而改善用户体验。

**关键要点**

遵循网站层级结构的最佳实践非常重要，因为这能让访客更容易找到所需内容，也能让搜索引擎更高效地抓取和索引你的网站。这些做法有助于提升用户导航体验、改善 SEO，并帮助搜索引擎理解网站各页面之间的关系。
